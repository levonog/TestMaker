Detailed Description

Supports creating directed and non-directed graphs.
Graph always connected (except the case, when number of edges is low than number of vertices)
Graph always fix a random tree, and try to deduce, are your Graph small or big (deduces by number of edges)
If your Graph have more that n*n/2 edges, where n is number of vertices, then Graph is big, otherwise small.
Big Graph generated this way: it takes full-connected Graph, and tries to remove random (u, v). If that edge is already deleted or that edge is in fixed tree, algorithm tries again. In worst case there is 50% chance to mistake, so number of trying should be ~ 2
Small Graph generated this way: it takes empty fraph, and add fixed tree edges. After that if there is any edges left tries to add random (u, v). If that edge is already added algorithm tries again. In worst case there is 50% chance to mistake, so number of trying should be ~ 2
If you want to generate not connected Graphs, you must use GraphMerger class

Here is an example for Graph. You can try it by taking it from Graph.h



int main()
{
PrimitiveTest<int>* vertices_count = CreateElement(3, 5);
PrimitiveTest<int>* edges_count = CreateElement(5, 10);
Graph* graph = new Graph(vertices_count, edges_count);
graph->Buckle()->Directed();
graph->PrintType(Graph::LIST_OF_EDGES);
graph->Generate();
graph->Print();
delete graph;
delete edges_count;

}

◆ PrintType()

virtual Graph* Graph::PrintType ( PRINT_TYPE  _print_type)
protectedvirtual

Specifies the way to output tha graph. For now available only 3 way to output:

  1. CONNECTION_MATRIX - count of vertices, if there is an edge (v, u), then matrix[v][u]=1:
    n
    0 1 ... 1
    0 1 ... 0
    ... ... ... ...
    1 0 ... 1

  2. CONNECTION_LIST - count of vertices, list of neighbors for each of vertex:

    n
    m1 v11 ... ... ... v1m1
    m2 v21 ... v2m2
    ... ... ... ...
    mn vn1 ... ... vnmn

  3. LIST_OF_EDGES - count of vertices, if there is an edge (v, u), then must be presented v u line:

    n
    v1 u1
    v2 u2
    ... ...
    vn un
